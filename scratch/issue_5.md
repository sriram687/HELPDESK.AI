## 🎯 Feature Description

Currently, our backend tries to load the SentenceTransformer model on start. If the server is offline or doesn't have Hugging Face API access, we degrade gracefully (as resolved by PR #60). 

However, to provide a fully robust production system, we want to run local ticket classification/embeddings generation *completely offline* on the server side using the ONNX Runtime! This will significantly reduce startup times and avoid large memory overheads.

---

## 🛠️ Requirements & Tech Stack

### 1. ONNX Model Export & Local Execution
* Export our SentenceTransformer model (e.g. `all-MiniLM-L6-v2`) to ONNX format.
* Setup local ONNX Runtime execution in `backend/services/onnx_service.py` to:
  * Parse text tokens.
  * Execute vector embeddings calculation offline.
  * Classify ticket category and priority without querying external Hugging Face APIs.

### 2. High-Performance Tokenizer
* Implement or integrate an offline Hugging Face tokenizer (using `tokenizers` library) inside the FastAPI backend.
* Create tests comparing the vectors generated by our ONNX model vs the PyTorch SentenceTransformer to ensure cosine similarity is close to 1.0!

---

## 🚀 Suggested Implementation Steps

1. **Add ONNX Dependencies**:
   * Add `onnxruntime` and `tokenizers` to `backend/requirements.txt`.
2. **Model ONNX Conversion Script**:
   * Write a small model converter script under `backend/scripts/convert_to_onnx.py`.
3. **Integrate Local Handler**:
   * Setup fallback logic: If model weight is present locally, execute via `onnxruntime` instantly.
4. **Performance Benchmark**:
   * Log startup memory usage and inference speed benchmarks comparing the legacy SentenceTransformer load vs the ONNX runtime pipeline!
